Transformações no Power Query

▼

📋 Tabela: vendas

1. Coluna data → clique com botão direito → Alterar Tipo → Data

2. Coluna valor_total → Alterar Tipo → Número Decimal Fixo (moeda)

3. Coluna valor_unitario → Alterar Tipo → Número Decimal Fixo

4. Filtre status: remova linhas com status "Cancelado" e "Devolvido" (queremos só vendas efetivas)

5. Renomeie a query para fVendas (f de fato)

Convenção de nomes: Tabelas fato começam com f (fVendas) e tabelas dimensão com d (dClientes). Isso é padrão de mercado e mostra maturidade no modelo.

📋 Tabela: clientes

1. Coluna data_cadastro → Alterar Tipo → Data

2. Coluna email → Remover esta coluna (dados sensíveis, não vai pro dashboard)

3. Renomeie para dClientes

📋 Tabela: produtos

1. Coluna preco e custo → Alterar Tipo → Número Decimal Fixo

2. Adicione Coluna Personalizada: nome = margem_pct, fórmula:

= Number.Round(([preco] - [custo]) / [preco] * 100, 1)

3. Renomeie para dProdutos

📋 Tabela: vendedores

1. Coluna data_admissao → Alterar Tipo → Data

2. Renomeie para dVendedores

📋 Tabela: metas

1. Crie Coluna Personalizada chamada data_meta:

= #date([ano], [mes], 1)

2. Altere o tipo de data_meta para Data

3. Remova colunas ano e mes (ficaram redundantes)

4. Renomeie para fMetas

🗓️ Criar Tabela Calendário

1. No Power Query: Nova Fonte → Consulta Nula

2. Na barra de fórmulas, cole:

= List.Dates(#date(2024,1,1), 730, #duration(1,0,0,0))

3. Clique em Para Tabela → OK

4. Renomeie a coluna para Data e altere para tipo Data

5. Adicione estas colunas via Adicionar Coluna → Coluna de Exemplo ou Coluna Personalizada:

// Coluna: Ano = Date.Year([Data]) // Coluna: Mês Num = Date.Month([Data]) // Coluna: Mês Nome = Date.MonthName([Data], "pt-BR") // Coluna: Mês Abrev = Text.Start(Date.MonthName([Data], "pt-BR"), 3) // Coluna: Trimestre = "Q" & Text.From(Date.QuarterOfYear([Data])) // Coluna: Dia da Semana = Date.DayOfWeekName([Data], "pt-BR") // Coluna: Ano-Mês = Date.ToText([Data], "yyyy-MM")

6. Renomeie a query para dCalendario

7. Clique em Fechar e Aplicar

Por que criar calendário manualmente? Porque CALENDARAUTO() às vezes falha e não gera colunas como mês-nome, trimestre, dia-da-semana. Fazendo no Power Query você tem controle total e impressiona na entrevista.

  • Transformei dClientes (tipo data, removeu email, nome)
  • Transformei dProdutos (tipos, margem_pct, nome)
  • Transformei dVendedores (tipo data, nome)
  • Transformei fMetas (data_meta, removeu ano/mes, nome)
  • Criei dCalendario com 7 colunas

🧮 Projeto 1 — Medidas DAX 15 Medidas

5

Criar as medidas DAX

▼

Para cada medida: vá em Report View → selecione a tabela fVendas → Nova Medida → cole a fórmula → Enter.

Crie uma pasta "Medidas" para organizá-las: clique na medida → Propriedades → Pasta de exibição → _Medidas

🟢 Básicas

Medida 1 — Faturamento
Faturamento = SUM( fVendas[valor_total] )
Medida 2 — Quantidade Vendida
Qtd Vendida = SUM( fVendas[quantidade] )
Medida 3 — Nº de Vendas
Nº Vendas = COUNTROWS( fVendas )
Medida 4 — Ticket Médio
Ticket Médio = DIVIDE( [Faturamento], [Nº Vendas], 0 )
Medida 5 — Clientes Únicos
Clientes Únicos = DISTINCTCOUNT( fVendas[cliente_id] )

🟡 Time Intelligence

Medida 6 — Faturamento Mês Anterior
Fat Mês Anterior = CALCULATE( [Faturamento], PREVIOUSMONTH( dCalendario[Data] ) )
Medida 7 — Variação MoM %
Var MoM % = DIVIDE( [Faturamento] - [Fat Mês Anterior], [Fat Mês Anterior], 0 )
Medida 8 — Faturamento YTD (Acumulado do Ano)
Fat YTD = TOTALYTD( [Faturamento], dCalendario[Data] )
Medida 9 — Faturamento Ano Anterior
Fat Ano Anterior = CALCULATE( [Faturamento], SAMEPERIODLASTYEAR( dCalendario[Data] ) )
Medida 10 — Variação YoY %
Var YoY % = DIVIDE( [Faturamento] - [Fat Ano Anterior], [Fat Ano Anterior], 0 )

🔴 Avançadas

Medida 11 — Meta Total
Meta = SUM( fMetas[meta] )
Medida 12 — % da Meta Atingido
% Meta = DIVIDE( [Faturamento], [Meta], 0 )
Medida 13 — % do Total Geral (para gráfico de composição)
% do Total = DIVIDE( [Faturamento], CALCULATE( [Faturamento], ALL( fVendas ) ), 0 )
Medida 14 — Ranking de Produto
Rank Produto = RANKX( ALL( dProdutos[nome] ), [Faturamento],, DESC, DENSE )
Medida 15 — Desconto Médio %
Desconto Médio % = AVERAGE( fVendas[desconto_pct] )

15 medidas. É tudo. Note que com essas 15 você cobre: KPIs básicos (1-5), comparação temporal (6-10) e análise avançada (11-15). Formate cada medida: clic direito → Formato → moeda para valores, percentual para %, número inteiro para contagens.

  • Criei medidas 1-5 (básicas)
  • Criei medidas 6-10 (time intelligence)
  • Criei medidas 11-15 (avançadas)
  • Formatei todas as medidas corretamente

🧮 Projeto 2 — Medidas DAX de RH 17 Medidas

12

Criar medidas de People Analytics

▼

🟢 Headcount & Estrutura

Medida 1 — Headcount Ativo
Headcount = COUNTROWS( FILTER( dFuncionarios, dFuncionarios[ativo] = 1 ) )
Medida 2 — Total de Funcionários (todos)
Total Funcionários = COUNTROWS( dFuncionarios )
Medida 3 — Salário Médio
Salário Médio = AVERAGE( dFuncionarios[salario] )
Medida 4 — Folha de Pagamento (mensal)
Folha Mensal = SUMX( FILTER( dFuncionarios, dFuncionarios[ativo] = 1 ), dFuncionarios[salario] )
Medida 5 — Tempo Médio na Empresa (meses)
Tempo Médio Empresa = AVERAGE( dFuncionarios[tempo_empresa_meses] )

🟡 Turnover

Medida 6 — Total Saídas
Total Saídas = SUM( fSaidas[saidas] )
Medida 7 — Total Contratações
Total Contratações = SUM( fContratacoes[contratacoes] )
Medida 8 — Taxa de Turnover %
Turnover % = DIVIDE( [Total Saídas], [Headcount], 0 )
Medida 9 — Saldo Líquido (contratações - saídas)
Saldo Líquido = [Total Contratações] - [Total Saídas]
Medida 10 — Vagas Abertas
Vagas Abertas = SUM( fContratacoes[vagas_abertas] )
Medida 11 — Taxa de Conversão de Vagas
Conversão Vagas % = DIVIDE( [Total Contratações], [Vagas Abertas], 0 )
Medida 12 — Tempo Médio para Contratar (dias)
Tempo Médio Contratação = AVERAGE( fContratacoes[tempo_medio_dias] )

🔴 Desempenho & Diversidade

Medida 13 — Nota Média Geral
Nota Média = AVERAGE( fAvaliacoes[nota_geral] )
Medida 14 — % Alto Desempenho (nota ≥ 4.0)
% Alto Desempenho = DIVIDE( COUNTROWS( FILTER( fAvaliacoes, fAvaliacoes[nota_geral] >= 4 ) ), COUNTROWS( fAvaliacoes ), 0 )
Medida 15 — % Feminino
% Feminino = DIVIDE( COUNTROWS( FILTER( dFuncionarios, dFuncionarios[genero] = "F" ) ), [Total Funcionários], 0 )
Medida 16 — Gap Salarial de Gênero
Salário Médio M = CALCULATE( AVERAGE( dFuncionarios[salario] ), dFuncionarios[genero] = "M" ) Salário Médio F = CALCULATE( AVERAGE( dFuncionarios[salario] ), dFuncionarios[genero] = "F" ) Gap Salarial % = DIVIDE( [Salário Médio M] - [Salário Médio F], [Salário Médio M], 0 )
Medida 17 — Índice de Retenção %
Retenção % = 1 - [Turnover %]

17 medidas de People Analytics! Gap salarial, turnover e retenção são métricas que CEOs cobram do RH ativamente. Ter essas medidas no portfólio mostra que você entende negócio, não só ferramenta.

  • Medidas 1-5 (headcount e estrutura)
  • Medidas 6-12 (turnover e recrutamento)
  • Medidas 13-17 (desempenho e diversidade)
  • Formatei todas (moeda, %, inteiro)
